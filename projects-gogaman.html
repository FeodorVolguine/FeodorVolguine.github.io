A physically based real-time rendering engine

Global illumination

Light bouncing is simulated using a voxel cone tracing algorithm. This solution to GI requires no baking - light sources and geometry are fully dynamic.


View code ›

Reflections

Cone tracing through the voxel structure enables reflections, but at a fraction of the cost of raytracing triangle geometry.


View code ›

Emissive geometry

In addition to point and directional lights, geometry can emit light through the emissive material property.

Depth of field

A physical camera model combined with a separable circular convolution yields a nearly ground truth bokeh.


View code ›

Physically based rendering

Direct lighting equations implement PBR princples. Disney's diffuse BRDF and GGX specular BRDF are used.

ACES filmic tonemapping

The mapping from high dynamic range values to low dynamic range mimicks a film camera response

Tools

Miscellaneous features

  • Bloom around high-luminance image regions using an iterative separable Gaussian convolution

  • Film grain using procedural noise which is modulated based on camera ISO parameter

  • Various tonemapping options

  • G-buffer albedo compression using edge-directed chrominance reconstruction
